# Hexwar — Milestone 03

## Goals

1. **Map-gen polish**: Fix river (neutral-band procedural winding, ~6-10 tiles) and resource nodes (per-match seed, balanced regional allocation, min spacing ≥2).
2. **Make buildings exist**: BuildManager — command centers, spawn buildings, placement rules, immediate cost + per-faction maintenance, CC build time (5s), rebuild cooldown (15s), auto-placed starting buildings (3 Infantry Barracks + 1 Tank Division per faction).
3. **Let the player build**: Hex-first context BuildPanel (select hex → show valid buildables → click to build; toggle spawn speed Off/Low/High).
4. **Tooltip UX refactor**: Bottom panel, 500ms hover delay, persists on selection, building info included, river line conditional.

## Design Decisions

1. Per-match random seed for map generation; seed logged/displayed for repro.
2. River: procedural winding path from left-neutral column (q = 5) to right-neutral column (q = 10), r drift {-1, 0, +1}, rare excursions into adjacent bands (friendly q=4, enemy q=11).
3. Resource nodes: proportional allocation per region (friendly/neutral/enemy), min hex distance ≥2, shuffled global pool so types spread.
4. Build UX: hex-first context panel. No separate placement-mode hover preview.
5. Tooltip: fixed bottom panel (not following mouse). Show after 500ms on same hex. Persist while hex selected. River info shown only when tile has river.

## Validation Plan

1. `tsc --noEmit` clean; `vite build` succeeds.
2. `npm run dev` → Start → GameScreen:
   - River thin, neutral-fronted, varies per match seed.
   - Nodes spread across regions, no clustering.
   - Top-bar maintenance drain > 0 at match start.
   - Same seed → same river + node layout.
3. Tooltip: bottom panel, 500ms hover delay, persists on selection, river line conditional.
4. Build:
   - Select player-owned hex with no CC → BuildPanel shows Command Center with cost.
   - Click to build → cost deducted, hex claimed, CC shows under-construction for ~5s then ready.
   - Ready CC + no building → BuildPanel shows 3 spawn types; build one → appears, maintenance rises.
   - Select building hex → toggle spawn speed Off→Low→High; top bar maintenance changes.
5. Verity friendly/enemy maintenance does NOT cross-drain.

## Risks

- Per-faction maintenance correctness: verify no cross-faction drain.
- River regeneration must clear old `river = true` tiles first.
- CC "ready" gating: spawn buildings blocked while CC under construction.
- Maintenance only registers once building is ready (not at placement).
- `canAfford` + `spend` atomic to avoid race with economy tick.
- BuildManager never owns display objects (state only).

## Out of Scope

- SpawnManager, units, combat, A*, pathfinding (M4-M5)
- Priority panel, notifications, win/lose (M6)
- Victory CC auto-infantry spawning (M4)
- CC destruction / damage / rebuild-cooldown trigger from combat (M5)
